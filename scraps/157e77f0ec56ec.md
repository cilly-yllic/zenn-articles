# 【メモ】あとで調べる・理解する・解決する
## 2024-02-01

```
type Key = 'hoge' | 'foo' | 'piyo'
type Obj<K extends Key> = {
    [key in K]: any[]
}
const exec = <T extends Key = Key>(key: T) => {
    const obj: Obj<T> = {
        [key]: []
    }
    console.log(obj)
}
```

上記コードで `const obj`の部分で  
`TS2322: Type { [x: string]: never[]; } is not assignable to type Obj<T>`  
のエラーが出てしまう。  
`[key as T]: []`としても変化なし

おそらくKeyはunion型で'hoge' | 'foo'などの複数を指定されている可能性があるにも関わらず、引数のkeyは一つしか渡されないので、その場合が考慮されてないでしょ的な状態になってしまうという事な気がする。  
しかし、union typeのうち一つという表現方法や、それ以外の解決方法がわからないので、保留。

```
const obj: Obj<T> = {
        [key]: []
} as Obj<T>
```

という力技はあまり応用が効かないので、できれば使いたくない。
